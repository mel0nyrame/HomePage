# 项目文档

## 项目概述

**项目名称**: homepage-main  
**描述**: 游戏展示官网后端  
**技术栈**: Spring Boot 3.5.14 + Spring Security 6 + OAuth2 Resource Server + MyBatis-Plus 3.5.16 + MySQL 8.4 + JWT (Nimbus) + SpringDoc OpenAPI  
**Java 版本**: 21  
**包名**: `com.homepage`

### 模块结构

```
homepage-main (父 POM)
├── homepage-common    — 公共模块：实体、DTO、JWT 工具、全局异常处理、响应封装、安全主体
└── homepage-auth      — 认证及业务模块：Controller、Service、Mapper、Security 配置、初始数据
```

**依赖关系**: `auth → common`

## 已实现功能

- 用户注册/登录（支持用户名或邮箱登录）
- 管理员登录（JWT scope 区分 USER / ADMIN 角色）
- 管理员注册（仅 ADMIN 角色可调用）
- JWT 令牌签发与验证（HS256 对称加密，2天过期）
- 基于角色的权限控制（`@PreAuthorize("hasRole('ADMIN')")`）
- 全局异常处理、统一响应格式
- OpenAPI 文档（SpringDoc + Swagger UI）
- Actuator 健康检查与监控端点
- MyBatis-Plus 自动填充（create_time / update_time）
- 首次启动自动创建初始管理员（`DataInitializer`）

## 业务逻辑

### 一、用户登录、注册、鉴权

方案：**Spring Security + OAuth2 Resource Server + Nimbus JWT + BCrypt**

---

#### 1. 注册流程

`POST /api/user/register`

```
客户端提交 nickname + username + email + password
  │
  ▼
UserController.register()
  │
  ▼
UserServiceImpl.register()
  ├── lambdaQuery().eq(UserEntity::getUsername, ...) → 已存在则 USER_ALREADY_EXIST (1002)
  ├── lambdaQuery().eq(UserEntity::getEmail, ...) → 已存在则 USER_EMAIL_EXIST (1009)
  ├── BCryptPasswordEncoder.encode(明文密码) → 密文
  └── this.save(user) 入库
```

注册只创建用户，不自动登录，客户端需单独调用登录接口。

---

#### 2. 登录流程

`POST /api/user/login`

```
客户端提交 account + password （account 可以是用户名或邮箱）
  │
  ▼
UserController.login()
  │
  ▼
UserServiceImpl.login()
  ├── 1. lambdaQuery().and(w -> w.eq(UserEntity::getUsername, account)
  │                                     .or().eq(UserEntity::getEmail, account))
  │        .exists() → 不存在则 USER_NOT_EXIST (1001)
  ├── 2. 构建 UsernamePasswordAuthenticationToken，提交给 userAuthenticationManager.authenticate()
  │     ├── DaoAuthenticationProvider 回调 loadUserByUsername() → 返回 HomepageUserDetails
  │     ├── BCryptPasswordEncoder.matches() 比对密码
  │     ├── 密码错误 → BadCredentialsException → USER_PASSWORD_ERROR (1003)
  │     ├── 账号禁用 → DisabledException → USER_ACCOUNT_DISABLED (1004)
  │     └── 认证通过 → 返回已认证的 Authentication 对象
  └── 3. JwtUtil.generateToken(authentication) → 返回 JWT 字符串
```

**Token 结构：**

| Claim | 值 | 说明 |
| --- | --- | --- |
| 算法 | HS256 | 对称加密，签发和验证用同一密钥 |
| `iss` | `"homepage"` | 签发者 |
| `iat` | 当前时间 | 签发时间 |
| `exp` | 当前时间 + 2天 | 过期时间 |
| `sub` | 用户名 | 令牌主体 |
| `scope` | 权限列表 | 如 `USER`、`ADMIN` |

---

#### 3. 鉴权流程

验证由 Spring Security 内置的 `BearerTokenAuthenticationFilter` 自动完成。

```
请求到达（Header: Authorization: Bearer <JWT>）
  │
  ▼
BearerTokenAuthenticationFilter
  ├── 1. 从 Authorization 头提取 Bearer token
  │     └── 无 token 或格式不对 → RestAuthenticationEntryPoint → 401
  ├── 2. NimbusJwtDecoder.decode(token)
  │     ├── 拆分 header.payload.signature，用 SecretKey 做 HMAC-SHA256 验签
  │     ├── 签名不匹配 → 401
  │     ├── 检查 exp，已过期 → 401
  │     └── 检查 nbf（若有）
  ├── 3. JwtAuthenticationConverter 将 scope claim 转为 GrantedAuthority
  │     └── "USER" → "ROLE_USER", "ADMIN" → "ROLE_ADMIN"
  └── 4. SecurityContextHolder 注入认证信息
  │
  ▼
请求到达 Controller，可通过以下方式获取用户：
  - SecurityContextHolder.getContext().getAuthentication()
  - @AuthenticationPrincipal Jwt jwt
  │
  ▼
权限不足 → RestAccessDeniedHandler → 403
```

**SecurityConfig 核心配置：**

```java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity          // 启用 @PreAuthorize 方法级权限注解
public class SecurityConfig {

    // 两个独立的 AuthenticationManager，各自绑定专用的 UserDetailsService

    @Primary
    @Bean("userAuthenticationManager")
    public AuthenticationManager userAuthenticationManager(
            UserService userDetailsService, PasswordEncoder passwordEncoder) {
        DaoAuthenticationProvider provider = new DaoAuthenticationProvider(userDetailsService);
        provider.setPasswordEncoder(passwordEncoder);
        return new ProviderManager(provider);
    }

    @Bean("adminAuthenticationManager")
    public AuthenticationManager adminAuthenticationManager(
            AdminService adminDetailsService, PasswordEncoder passwordEncoder) {
        DaoAuthenticationProvider provider = new DaoAuthenticationProvider(adminDetailsService);
        provider.setPasswordEncoder(passwordEncoder);
        return new ProviderManager(provider);
    }

    // JWT scope → ROLE_ 前缀转换
    @Bean
    public JwtAuthenticationConverter jwtAuthenticationConverter() {
        JwtGrantedAuthoritiesConverter gac = new JwtGrantedAuthoritiesConverter();
        gac.setAuthoritiesClaimName("scope");
        gac.setAuthorityPrefix("ROLE_");
        JwtAuthenticationConverter converter = new JwtAuthenticationConverter();
        converter.setJwtGrantedAuthoritiesConverter(gac);
        return converter;
    }

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http,
                                                    JwtAuthenticationConverter jwtAuthenticationConverter) {
        http
            .csrf(AbstractHttpConfigurer::disable)
            .sessionManagement(sm -> sm.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .oauth2ResourceServer(oauth2 -> oauth2
                .jwt(jwt -> jwt.jwtAuthenticationConverter(jwtAuthenticationConverter)))
            .exceptionHandling(ex -> ex
                .authenticationEntryPoint(new RestAuthenticationEntryPoint())   // 401
                .accessDeniedHandler(new RestAccessDeniedHandler()))            // 403
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/user/login", "/api/user/register",
                                 "/api/admin/login", "/swagger-ui.html",
                                 "/swagger-ui/**", "/api-docs/**",
                                 "/api-docs", "/actuator/**").permitAll()
                .anyRequest().authenticated());
        return http.build();
    }
}
```

**JWT 编解码配置（JwtConfig）：**

```java
@Bean
public JwtEncoder jwtEncoder() {
    SecretKey key = new SecretKeySpec(JWT_SECRET.getBytes(StandardCharsets.UTF_8), "HmacSHA256");
    return new NimbusJwtEncoder(new ImmutableSecret<>(key));
}

@Bean
public JwtDecoder jwtDecoder() {
    return NimbusJwtDecoder
        .withSecretKey(new SecretKeySpec(JWT_SECRET.getBytes(StandardCharsets.UTF_8), "HmacSHA256"))
        .build();
}
```

### 二、管理员登录、注册

管理员与用户使用**独立**的 `AuthenticationManager` 和 `UserDetailsService`，通过 JWT scope 区分角色。

---

#### 1. 管理员登录

`POST /api/admin/login`

```
客户端提交 account + password
  │
  ▼
AdminController.login()
  │
  ▼
AdminServiceImpl.login()
  ├── 1. lambdaQuery().eq(AdminEntity::getAccount, account).one() → 不存在则 USER_NOT_EXIST (1001)
  ├── 2. 提交给 adminAuthenticationManager.authenticate()
  │     ├── DaoAuthenticationProvider 回调 AdminServiceImpl.loadUserByUsername()
  │     │     └── 返回 AdminUserDetails（authorities = "ADMIN"）
  │     ├── 密码错误 → BadCredentialsException → USER_PASSWORD_ERROR (1003)
  │     ├── 账号禁用 → DisabledException → USER_ACCOUNT_DISABLED (1004)
  │     └── 认证通过 → 返回 Authentication
  └── 3. JwtUtil.generateToken(authentication) → JWT scope = "ADMIN"
```

#### 2. 管理员注册（需 ADMIN 权限）

`POST /api/admin/register`  
权限要求：`@PreAuthorize("hasRole('ADMIN')")`

```
客户端提交 account + password（Header 需携带 ADMIN 的 JWT）
  │
  ▼
AdminController.register()
  │
  ▼
AdminServiceImpl.register()
  ├── lambdaQuery().eq(AdminEntity::getAccount, account).exists() → 已存在则 USER_ALREADY_EXIST (1002)
  ├── BCryptPasswordEncoder.encode(明文密码) → 密文
  └── this.save(adminEntity) 入库
```

#### 3. 角色权限机制

- User 登录 → JWT scope = `USER` → `JwtAuthenticationConverter` → `ROLE_USER`
- Admin 登录 → JWT scope = `ADMIN` → `JwtAuthenticationConverter` → `ROLE_ADMIN`
- 授权注解：`@PreAuthorize("hasRole('ADMIN')")` 检查 `ROLE_ADMIN`
- `@EnableMethodSecurity` 必须显式开启，否则 `@PreAuthorize` 不生效

#### 4. 初始管理员

application.yml 中配置了初始管理员凭据：

```yaml
app:
  admin:
    init-account: admin
    init-password: password
```

（`DataInitializer` 通过 `CommandLineRunner` 在首次启动时自动创建该管理员，密码经 BCrypt 加密后入库）

---

### 三、实体与 Spring Security 解耦设计

实体类（`AdminEntity`、`UserEntity`）不再直接实现 `UserDetails`，改为纯数据库映射 POJO，避免了 MyBatis-Plus 属性类型推断与 `UserDetails.getAuthorities()` 返回 `Collection` 类型的冲突。

安全主体由独立的包装类实现：

```
AdminEntity (纯 POJO, @TableName("homepage_admin"))
    └── AdminUserDetails (实现 UserDetails, 包装 AdminEntity)

UserEntity (纯 POJO, @TableName("homepage_user"))
    └── HomepageUserDetails (实现 UserDetails, 包装 UserEntity)
```

Service 层 `loadUserByUsername()` 从数据库查出实体后，包装为对应的安全主体返回。
