# 项目文档

## 项目描述

该项目为展示自制游戏想法的交流平台和官网

## 实现功能

- 需要实现的前端功能
  - 渲染md文档格式

- 需要实现的后端功能
  - 用户登陆，登出
  - 论坛交流，发布帖子，给帖子打标签
  - 帖子浏览量、点赞量、曝光量
  - 搜索帖子，支持模糊搜索，过滤搜索
  - 用户私信
  - 用户更换用户名，邮箱，查看自己发过的帖子
  - 拉黑用户

## 业务逻辑

### 用户登陆、注册、鉴权

方案：**Spring Security + OAuth2 Resource Server + NimbusJWT + BCrypt**

---

#### 注册流程

`POST /api/user/register`

```text
客户端提交 username + password + nickname
  │
  ▼
UserController.register()
  │
  ▼
UserServiceImpl.register()
  ├── userMapper.existsByUsername(username) → 已存在则返回 USER_ALREADY_EXIST (1002)
  ├── BCryptPasswordEncoder.encode(明文密码) → 密文
  └── userMapper.insertUser(user) 入库
```

注册只创建用户，不自动登录，客户端需单独调用登录接口。

---

#### 登录流程

`POST /api/user/login`

```text
客户端提交 username + password
  │
  ▼
UserController.login()
  │
  ▼
UserServiceImpl.login()
  ├── 1. userMapper.existsByUsername(username) → 不存在则返回 USER_NOT_EXIST (1001)
  ├── 2. 构建 UsernamePasswordAuthenticationToken，提交给 AuthenticationManager.authenticate()
  │     ├── Spring Security 内部回调 loadUserByUsername() 从数据库查出用户
  │     ├── DaoAuthenticationProvider 用 BCryptPasswordEncoder.matches() 比对密码
  │     ├── 密码错误 → BadCredentialsException → USER_PASSWORD_ERROR (1003)
  │     ├── 账号禁用 → DisabledException → USER_ACCOUNT_DISABLED (1004)
  │     └── 认证通过 → 返回已认证的 Authentication 对象
  └── 3. JwtUtil.generateToken(authentication) → 返回 JWT 字符串
```

**Token 结构：**

| Claim | 值 | 说明 |
| --- | --- | --- |
| 算法 | HS256 | 对称加密，签发和验证用同一密钥 |
| `iss` | `"homepage"` | 签发者 |
| `iat` | 当前时间 | 签发时间 |
| `exp` | 当前时间 + 2天 | 过期时间 |
| `sub` | 用户名 | 令牌主体 |
| `scope` | 权限列表 | 如 `USER`、`ADMIN` |

#### 鉴权流程

验证由 Spring Security 内置的 `BearerTokenAuthenticationFilter` 自动完成，无需自定义拦截器。

```text
请求到达（Header: Authorization: Bearer <JWT>）
  │
  ▼
BearerTokenAuthenticationFilter
  ├── 1. 从 Authorization 头提取 Bearer token
  │     └── 无 token 或格式不对 → 401
  ├── 2. NimbusJwtDecoder.decode(token)
  │     ├── 拆分 header.payload.signature，用 SecretKey 做 HMAC-SHA256 验签
  │     ├── 签名不匹配 → 401
  │     ├── 检查 exp，已过期 → 401
  │     └── 检查 nbf（若有）
  ├── 3. 构建 JwtAuthenticationToken，权限从 scope claim 解析
  └── 4. SecurityContextHolder 注入认证信息
  │
  ▼
请求到达 Controller，可通过以下方式获取用户：
  - SecurityContextHolder.getContext().getAuthentication()
  - @AuthenticationPrincipal Jwt jwt
  - @AuthenticationPrincipal UserDetails userDetails
  │
  ▼
权限不足 → BearerTokenAccessDeniedHandler → 403
```

**SecurityConfig 核心配置：**

```java
http
    .csrf(AbstractHttpConfigurer::disable)          // JWT 无需 CSRF 防护
    .sessionManagement(session ->
        session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))  // 无状态
    .oauth2ResourceServer(oauth2 ->
        oauth2.jwt(Customizer.withDefaults()))       // 启用 JWT 资源服务器
    .exceptionHandling(exceptions -> exceptions
        .authenticationEntryPoint(new BearerTokenAuthenticationEntryPoint())  // 401
        .accessDeniedHandler(new BearerTokenAccessDeniedHandler()))           // 403
    .authorizeHttpRequests(authorize -> authorize
        .requestMatchers("/api/user/login", "/api/user/register").permitAll()
        .anyRequest().authenticated());
```

**JWT 编解码配置：**

```java
// JwtConfig — 签发和验证共用同一个 SecretKey
@Bean
public JwtEncoder jwtEncoder() {
    SecretKey key = new SecretKeySpec(JWT_SECRET.getBytes(), "HmacSHA256");
    return new NimbusJwtEncoder(new ImmutableSecret<>(key));
}

@Bean
public JwtDecoder jwtDecoder() {
    return NimbusJwtDecoder
        .withSecretKey(new SecretKeySpec(JWT_SECRET.getBytes(), "HmacSHA256"))
        .build();
}
```
